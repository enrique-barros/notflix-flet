main.py

El punto de arranque de la aplicación. Es el primer archivo que ejecuta Python y el único que llama a ft.run().

El código completo

main.py
import flet as ft
from config.theme import COLOR_BG
from router import setup_router


def main(page: ft.Page) -> None:
    page.title = "Notflix"
    page.padding = 0
    page.bgcolor = COLOR_BG
    page.horizontal_alignment = ft.CrossAxisAlignment.STRETCH
    setup_router(page)


if __name__ == "__main__":
    ft.run(main)

¿Cómo arranca Flet?

ft.run(main) lanza el motor de Flet. Flet crea la ventana, inicializa el entorno gráfico y llama a la función main pasándole un objeto page. Ese objeto representa la ventana completa de la app y es el que se pasa de función en función por todo el proyecto.

La función main(page)

Recibe el objeto page de Flet y configura las propiedades globales de la ventana antes de que aparezca nada en pantalla:

PropiedadValorQué hace
page.title"Notflix"Título de la ventana del sistema operativo
page.padding0Elimina el margen por defecto para ocupar toda la pantalla
page.bgcolorCOLOR_BGColor de fondo global (#141414)
page.horizontal_alignmentSTRETCHLos hijos se estiran para ocupar todo el ancho

if __name__ == "__main__"

Esta condición es un estándar de Python. El intérprete asigna la variable __name__ al valor "__main__" solo cuando el archivo se ejecuta directamente (ej: python main.py). Si otro archivo importa main.py, __name__ valdría "main" y ft.run() no se ejecutaría. Así se evita que la app arranque por accidente al ser importada.

Flujo completo de arranque

python main.py

Python ejecuta el archivo. La condición __name__ == "__main__" es verdadera.

ft.run(main)

Flet crea la ventana y llama a main(page) con el objeto página.

Configuración de la página

Se aplican título, fondo, padding y alineación.

setup_router(page)

El router toma el control, registra el handler de atrás y muestra la pantalla de perfiles.

La app está viva

Flet entra en el bucle de eventos y espera interacción del usuario.
