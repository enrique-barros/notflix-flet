Componentes visuales

Piezas visuales reutilizables. Reciben datos por parámetro y devuelven un widget de Flet. No saben nada de navegación ni del estado global de la app.

movie_card.py

Para qué sirve: La tarjeta tipo póster que aparece en las filas del catálogo. Muestra un gradiente de fondo único por película, un ícono decorativo, y un overlay con el título y los metadatos.

components/movie_card.py — build_movie_card()
def build_movie_card(movie: Movie, width: float = 180) -> ft.Container:
    height = width * 1.5          # ratio 2:3 tipo póster
    type_color = COLOR_PRIMARY if movie.type == "Movie" else "#0070f3"
    grad_colors = _poster_gradient(movie.title)

    poster = ft.Container(
        width=width,
        height=height,
        border_radius=RADIUS_CARD,
        gradient=ft.LinearGradient(
            begin=ft.Alignment(0, -1),   # arriba
            end=ft.Alignment(0, 1),    # abajo
            colors=grad_colors,
        ),
        content=ft.Stack(controls=[
            # 1. ícono decorativo de fondo
            ft.Container(
                alignment=ft.Alignment(0, -0.2),
                content=ft.Icon(ft.Icons.MOVIE_OUTLINED,
                    color=ft.Colors.with_opacity(0.25, "white"), size=48),
            ),
            # 2. overlay inferior con gradiente negro + texto
            ft.Container(
                alignment=ft.Alignment(0, 1),
                content=ft.Container(
                    gradient=ft.LinearGradient(
                        colors=[ft.Colors.with_opacity(0, "black"),
                                ft.Colors.with_opacity(0.85, "black")],
                    ),
                    content=ft.Column(controls=[
                        # badge, título, año·rating, géneros
                    ]),
                ),
            ),
        ]),
    )
    return poster

¿Qué es ft.Stack?

Un Stack apila sus hijos unos encima de otros, como capas. El primero queda abajo, el último arriba. Aquí se usa para poner el ícono decorativo detrás y el overlay con texto delante.

El gradiente por hash

components/movie_card.py — _poster_gradient()
_POSTER_GRADIENTS = [
    ["#0f0c29", "#302b63", "#24243e"],
    ["#1a1a2e", "#16213e", "#0f3460"],
    # … 10 combinaciones más
]

def _poster_gradient(title: str) -> list[str]:
    idx = hash(title) % len(_POSTER_GRADIENTS)
    return _POSTER_GRADIENTS[idx]

hash(title) convierte el título en un número entero. El operador % (módulo) devuelve el resto de dividir ese número entre la cantidad de gradientes, garantizando un índice siempre válido. El mismo título siempre da el mismo hash, por eso cada película tiene siempre el mismo color.

El guión bajo en _poster_gradient es una convención de Python que indica que la función es privada: solo debe usarse dentro de este archivo.

¿Qué es una callback?

El parámetro on_ver_detalle es una función que le pasa quien llama al componente. El banner no sabe cómo funciona la navegación — simplemente llama a esa función cuando el usuario pulsa un botón. Esto mantiene el componente desacoplado del resto de la app.

En home_view.py, quien llama al banner define esa función así:

home_view.py — cómo se pasa la callback
def ver_detalle(e: ft.ControlEvent) -> None:
    navigate_to(page, f"/detail/{featured.show_id}")

build_hero_banner(featured, on_ver_detalle=ver_detalle)

ink=True

Activa el efecto de onda (ripple) al pulsar el contenedor. Es el efecto visual que da feedback al usuario de que ha pulsado algo.

Truncar la descripción

movie.description[:160] coge los primeros 160 caracteres. Si la descripción es más larga, añade "…". Es una forma concisa de hacer un truncado condicional en una sola línea.

logo.py

Para qué sirve: Devuelve el texto "NOTFLIX" con el estilo de la marca. El componente más simple del proyecto.
components/logo.py
def build_logo(texto: str = "NotFlix") -> ft.Container:
    logo = ft.Text(
        texto.upper(),
        size=FONT_SIZE_LOGO,
        weight=ft.FontWeight.BOLD,
        color=COLOR_PRIMARY,
    )
    return ft.Container(
        bgcolor=ft.Colors.SURFACE_CONTAINER_HIGHEST,
        padding=PADDING_LOGO,
        border_radius=RADIUS_CARD,
        alignment=ft.Alignment.CENTER,
        content=logo,
    )

El parámetro texto tiene un valor por defecto ("NotFlix"), así que se puede llamar sin argumentos: build_logo(). En esta versión del proyecto apenas se usa porque el logo del AppBar se define directamente en el router.
