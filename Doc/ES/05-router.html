Router

El cerebro de la navegación. Controla qué pantalla se muestra y gestiona el historial. Es el archivo que conecta todas las vistas.

Cómo funciona la navegación en Flet

Flet no usa URLs reales como un navegador. En su lugar, mantiene una pila de Views. Piensa en ella como una pila de cartas: la carta de arriba es lo que el usuario ve. Cuando navegas hacia adelante, se añade una carta encima. Cuando pulsas atrás, se elimina la carta de arriba.

cómo funciona la pila de Views
# En la pantalla de inicio:
page.views = [profile_view]    # el usuario ve profile

# Al ir al home:
page.views = [profile_view, home_view]    # el usuario ve home

# Al ver un detalle:
page.views = [profile_view, home_view, detail_view]    # el usuario ve detail

# Al pulsar atrás:
page.views.pop()    # elimina detail → el usuario ve home

La base siempre es profile_view. Aunque el usuario nunca vuelva a ella haciendo clic, sigue estando en la pila por si pulsa el botón físico de atrás varias veces.

El índice de películas — _MOVIE_INDEX

router.py — índice de películas
_MOVIE_INDEX: dict[str, Movie] = {m.show_id: m for m in load_movies()}

Un diccionario que mapea show_id → Movie. Se construye una sola vez al importar el módulo, con una dict comprehension (igual que list comprehension pero crea un diccionario).

Cuando la ruta es /detail/s123, se extrae s123 y se hace _MOVIE_INDEX.get("s123"). Buscar en un diccionario es instantáneo independientemente de cuántas películas haya — mucho más eficiente que recorrer una lista entera.

navigate_to() — la función clave

router.py — navigate_to()
def navigate_to(page: ft.Page, route: str) -> None:
    page.views.clear()                     # borra toda la pila
    page.views.append(_view_profiles(page))  # siempre la base

    if route == "/home":
        page.views.append(_view_home(page))

    elif route.startswith("/detail/"):
        show_id = route.split("/detail/", 1)[-1]   # extrae "s123"
        page.views.append(_view_home(page))
        page.views.append(_view_detail(page, show_id))

    page.update()   # redibuja

¿Por qué limpiar y reconstruir en vez de solo añadir?

Si solo añadieras, la pila crecería infinitamente al navegar entre pantallas. Limpiar y reconstruir garantiza que la pila siempre tiene exactamente la profundidad correcta para la ruta actual.

Extraer el show_id de la ruta

route.split("/detail/", 1) parte la cadena "/detail/s123" en máximo 2 partes: ["", "s123"]. El [-1] coge el último elemento, que es el ID. El 1 limita los cortes para no romper IDs que pudieran contener el texto "/detail/".

Las funciones constructoras de Views

Cada pantalla se envuelve en un ft.View, que es la unidad de pantalla de Flet. Aquí se define la ruta, el color de fondo y, opcionalmente, la AppBar.

router.py — _view_home()
def _view_home(page: ft.Page) -> ft.View:
    return ft.View(
        route="/home",
        bgcolor=COLOR_BG,
        appbar=ft.AppBar(
            title=ft.Text("NOTFLIX", color="#E50914", weight=ft.FontWeight.BOLD),
            bgcolor="#0a0a0a",
            automatically_imply_leading=False,  # sin botón atrás automático
        ),
        controls=[
            ft.Container(
                expand=True,
                padding=ft.Padding(16, 0, 16, 16),
                content=ft.SafeArea(expand=True, content=build_home_view(page)),
            )
        ],
    )

automatically_imply_leading=False desactiva el botón de flecha atrás que Flet pondría automáticamente en el AppBar cuando hay más de una view en la pila. En el home no se quiere ese botón.

ft.SafeArea añade padding automático para evitar que el contenido quede tapado por notches, cámaras o barras del sistema en móvil.

setup_router() — el arranque

router.py — setup_router()
def setup_router(page: ft.Page) -> None:
    def view_pop(e: ft.ViewPopEvent) -> None:
        if len(page.views) > 1:
            page.views.pop()   # elimina la última view
            page.update()

    page.on_view_pop = view_pop    # botón físico de atrás
    navigate_to(page, "/")         # pantalla inicial

page.on_view_pop es el evento que Flet dispara cuando el usuario pulsa el botón físico de atrás (en Android) o el gesto de atrás. La función view_pop elimina la última view de la pila. El len > 1 evita eliminar la view base y quedarse con una pila vacía.

Esta función se llama una sola vez desde main.py al arrancar la app.
