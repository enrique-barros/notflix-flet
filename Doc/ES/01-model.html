Modelo de datos

Define cómo se representa un título de Netflix dentro del programa y carga todos los datos desde el CSV. Es la base de toda la app — sin esto, no hay nada que mostrar.

Por qué existe este archivo

La app necesita trabajar con datos de películas. Esos datos vienen de un CSV con miles de filas. Este archivo hace dos cosas: define cómo se ve una película dentro del programa (la clase Movie), y proporciona la función que lee el CSV y convierte cada fila en un objeto Movie.

Es el único archivo que toca el disco. Todo lo demás trabaja con listas de objetos Movie que este archivo ha creado. Eso es separación de responsabilidades.

El archivo CSV

Un CSV es un archivo de texto donde cada línea es una fila y los valores están separados por comas. El archivo netflix_titles.csv tiene una cabecera con los nombres de columna y luego una fila por cada título:

netflix_titles.csv (extracto) 
show_id,type,title,director,cast,country,date_added,...
s1,Movie,Dick Johnson Is Dead,Kirsten Johnson,,United States,...
s2,TV Show,Blood & Water,,Ama Qamata,...,South Africa,...

Python puede leer esto con csv.DictReader, que convierte cada fila en un diccionario donde las claves son los nombres de columna. Mucho más cómodo que trabajar con índices numéricos.

La clase Movie — @dataclass

Un dataclass es una forma de crear clases que solo guardan datos, sin tener que escribir el método __init__ a mano. Python lo genera automáticamente.

movie_model.py — clase Movie 
from dataclasses import dataclass

@dataclass
class Movie:
    show_id: str       # identificador único, ej: "s1"
    type: str          # "Movie" o "TV Show"
    title: str
    director: str
    cast: str          # actores separados por coma
    country: str
    date_added: str
    release_year: str
    rating: str        # "PG-13", "TV-MA"…
    duration: str       # "90 min" o "2 Seasons"
    listed_in: str     # géneros separados por coma
    description: str

    @property
    def genres(self) -> list[str]:
        # "Drama, Comedy, Action" → ["Drama", "Comedy", "Action"]
        return [g.strip() for g in self.listed_in.split(",") if g.strip()]

¿Qué es @property?

genres es una propiedad calculada: no se guarda en el objeto, se calcula cada vez que se accede. El CSV tiene listed_in = "Drama, Comedy" como texto. La propiedad parte ese texto por coma y devuelve una lista limpia. Se usa como movie.genres, sin paréntesis, igual que si fuera un atributo normal.

Clave: .strip() elimina los espacios en blanco de cada género. Sin esto, "Drama" y " Drama" serían géneros distintos.

La función load_movies()

Lee el CSV completo y devuelve una lista con todos los objetos Movie válidos.

movie_model.py — load_movies() 
from pathlib import Path
import csv

DATA_PATH = Path(__file__).parent.parent / "data" / "netflix_titles.csv"

def load_movies(path: Path = DATA_PATH) -> list[Movie]:
    movies: list[Movie] = []

    with open(path, encoding="utf-8") as f:
        reader = csv.DictReader(f)

        for row in reader:
            title = row.get("title", "").strip()
            type_ = row.get("type", "").strip()

            if not title or not type_:
                continue        # salta filas incompletas

            movies.append(Movie(
                show_id=row.get("show_id", ""),
                type=type_,
                title=title,
                # … resto de campos
            ))

    return movies

¿Qué es DATA_PATH?

Path(__file__) es la ruta del archivo actual (movie_model.py). .parent.parent sube dos carpetas hasta la raíz del proyecto. Luego se concatena "data/netflix_titles.csv". Así la ruta funciona independientemente de desde dónde ejecutes la app.

¿Por qué usar .get() en vez de row["key"]?

Si la clave no existe en el diccionario, row["key"] lanza un error. row.get("key", "") devuelve una cadena vacía si no existe. Como el CSV puede tener filas con campos vacíos, esto evita que la app explote con datos sucios.

¿Por qué omitir filas sin título o tipo?

Algunos CSVs tienen filas vacías o corruptas al final. Si title o type están vacíos, ese objeto Movie sería inútil para la app. continue salta esa iteración y pasa a la siguiente fila.
