main.py

The application's entry point. It's the first file Python runs and the only one that calls ft.run().

The full code

main.py
import flet as ft
from config.theme import COLOR_BG
from router import setup_router


def main(page: ft.Page) -> None:
    page.title = "Notflix"
    page.padding = 0
    page.bgcolor = COLOR_BG
    page.horizontal_alignment = ft.CrossAxisAlignment.STRETCH
    setup_router(page)


if __name__ == "__main__":
    ft.run(main)

How Flet starts up

ft.run(main) launches the Flet engine. Flet creates the window, initializes the graphics environment, and calls the main function passing it a page object. That object represents the app's full window and is the one passed from function to function throughout the project.

The main(page) function

It receives the page object from Flet and configures the window's global properties before anything appears on screen:

PropertyValueWhat it does
page.title"Notflix"Title of the operating system window
page.padding0Removes the default margin to fill the full screen
page.bgcolorCOLOR_BGGlobal background color (#141414)
page.horizontal_alignmentSTRETCHChildren stretch to fill the full width

if __name__ == "__main__"

This condition is a Python standard. The interpreter assigns the variable __name__ the value "__main__" only when the file is run directly (e.g. python main.py). If another file imports main.py, __name__ would be "main" and ft.run() would not execute. This prevents the app from starting accidentally when imported.

Full startup flow

python main.py

Python runs the file. The __name__ == "__main__" condition is true.

ft.run(main)

Flet creates the window and calls main(page) with the page object.

Page configuration

Title, background, padding, and alignment are applied.

setup_router(page)

The router takes over, registers the back handler, and shows the profile screen.

The app is alive

Flet enters the event loop and waits for user interaction.
