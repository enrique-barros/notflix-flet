Data model

Defines how a Netflix title is represented inside the program and loads all data from the CSV. It's the foundation of the entire app — without this, there's nothing to show.

Why this file exists

The app needs to work with movie data. That data comes from a CSV with thousands of rows. This file does two things: it defines what a movie looks like inside the program (the Movie class), and provides the function that reads the CSV and converts each row into a Movie object.

It's the only file that touches the disk. Everything else works with lists of Movie objects that this file has created. That's separation of concerns.

The CSV file

A CSV is a text file where each line is a row and the values are separated by commas. The netflix_titles.csv file has a header with column names followed by one row per title:

netflix_titles.csv (excerpt) 
show_id,type,title,director,cast,country,date_added,...
s1,Movie,Dick Johnson Is Dead,Kirsten Johnson,,United States,...
s2,TV Show,Blood & Water,,Ama Qamata,...,South Africa,...

Python can read this with csv.DictReader, which converts each row into a dictionary where the keys are column names. Much more convenient than working with numeric indices.

The Movie class — @dataclass

A dataclass is a way to create classes that only store data, without having to write the __init__ method by hand. Python generates it automatically.

movie_model.py — Movie class 
from dataclasses import dataclass

@dataclass
class Movie:
    show_id: str       # unique identifier, e.g. "s1"
    type: str          # "Movie" or "TV Show"
    title: str
    director: str
    cast: str          # comma-separated actors
    country: str
    date_added: str
    release_year: str
    rating: str        # "PG-13", "TV-MA"…
    duration: str       # "90 min" or "2 Seasons"
    listed_in: str     # comma-separated genres
    description: str

    @property
    def genres(self) -> list[str]:
        # "Drama, Comedy, Action" → ["Drama", "Comedy", "Action"]
        return [g.strip() for g in self.listed_in.split(",") if g.strip()]

What is @property?

genres is a computed property: it isn't stored in the object, it's calculated each time it's accessed. The CSV has listed_in = "Drama, Comedy" as a string. The property splits that string by comma and returns a clean list. It's used as movie.genres, without parentheses, just like a normal attribute.

Key point: .strip() removes whitespace from each genre. Without this, "Drama" and " Drama" would be treated as different genres.

The load_movies() function

Reads the full CSV and returns a list of all valid Movie objects.

movie_model.py — load_movies() 
from pathlib import Path
import csv

DATA_PATH = Path(__file__).parent.parent / "data" / "netflix_titles.csv"

def load_movies(path: Path = DATA_PATH) -> list[Movie]:
    movies: list[Movie] = []

    with open(path, encoding="utf-8") as f:
        reader = csv.DictReader(f)

        for row in reader:
            title = row.get("title", "").strip()
            type_ = row.get("type", "").strip()

            if not title or not type_:
                continue        # skip incomplete rows

            movies.append(Movie(
                show_id=row.get("show_id", ""),
                type=type_,
                title=title,
                # … rest of fields
            ))

    return movies

What is DATA_PATH?

Path(__file__) is the path of the current file (movie_model.py). .parent.parent goes up two folders to the project root. Then "data/netflix_titles.csv" is appended. This way the path works regardless of where you run the app from.

Why use .get() instead of row["key"]?

If the key doesn't exist in the dictionary, row["key"] raises an error. row.get("key", "") returns an empty string if it doesn't exist. Since the CSV may have rows with empty fields, this prevents the app from crashing on dirty data.

Why skip rows without a title or type?

Some CSVs have empty or corrupted rows at the end. If title or type are empty, that Movie object would be useless to the app. continue skips that iteration and moves to the next row.
