Visual components

Reusable visual pieces. They receive data as parameters and return a Flet widget. They know nothing about navigation or the app's global state.

movie_card.py

What it does: The poster-style card that appears in the catalog rows. It shows a unique gradient background per movie, a decorative icon, and an overlay with the title and metadata.

build_movie_card(movie, width)

components/movie_card.py — build_movie_card()
def build_movie_card(movie: Movie, width: float = 180) -> ft.Container:
    height = width * 1.5          # 2:3 poster ratio
    type_color = COLOR_PRIMARY if movie.type == "Movie" else "#0070f3"
    grad_colors = _poster_gradient(movie.title)

    poster = ft.Container(
        width=width,
        height=height,
        border_radius=RADIUS_CARD,
        gradient=ft.LinearGradient(
            begin=ft.Alignment(0, -1),   # top
            end=ft.Alignment(0, 1),    # bottom
            colors=grad_colors,
        ),
        content=ft.Stack(controls=[
            # 1. decorative background icon
            ft.Container(
                alignment=ft.Alignment(0, -0.2),
                content=ft.Icon(ft.Icons.MOVIE_OUTLINED,
                    color=ft.Colors.with_opacity(0.25, "white"), size=48),
            ),
            # 2. bottom overlay with black gradient + text
            ft.Container(
                alignment=ft.Alignment(0, 1),
                content=ft.Container(
                    gradient=ft.LinearGradient(
                        colors=[ft.Colors.with_opacity(0, "black"),
                                ft.Colors.with_opacity(0.85, "black")],
                    ),
                    content=ft.Column(controls=[
                        # badge, title, year·rating, genres
                    ]),
                ),
            ),
        ]),
    )
    return poster

What is ft.Stack?

A Stack layers its children on top of each other, like layers. The first one sits at the bottom, the last on top. Here it's used to place the decorative icon behind and the text overlay in front.

The hash-based gradient

components/movie_card.py — _poster_gradient()
_POSTER_GRADIENTS = [
    ["#0f0c29", "#302b63", "#24243e"],
    ["#1a1a2e", "#16213e", "#0f3460"],
    # … 10 more combinations
]

def _poster_gradient(title: str) -> list[str]:
    idx = hash(title) % len(_POSTER_GRADIENTS)
    return _POSTER_GRADIENTS[idx]

hash(title) converts the title into an integer. The % (modulo) operator returns the remainder of dividing that number by the number of gradients, guaranteeing a valid index every time. The same title always produces the same hash, so each movie always gets the same color.

The leading underscore in _poster_gradient is a Python convention indicating the function is private: it should only be used within this file.

What is a callback?

The on_ver_detalle parameter is a function passed in by whoever calls the component. The banner doesn't know how navigation works — it simply calls that function when the user presses a button. This keeps the component decoupled from the rest of the app.

In home_view.py, the caller defines that function like this:

home_view.py — how the callback is passed
def ver_detalle(e: ft.ControlEvent) -> None:
    navigate_to(page, f"/detail/{featured.show_id}")

build_hero_banner(featured, on_ver_detalle=ver_detalle)

ink=True

Enables the ripple effect when the container is tapped. It's the visual feedback that tells the user they've pressed something.

Truncating the description

movie.description[:160] takes the first 160 characters. If the description is longer, it appends "…". A concise way to do a conditional truncation in a single line.

logo.py

What it does: Returns the "NOTFLIX" text styled as the brand. The simplest component in the project.
components/logo.py
def build_logo(texto: str = "NotFlix") -> ft.Container:
    logo = ft.Text(
        texto.upper(),
        size=FONT_SIZE_LOGO,
        weight=ft.FontWeight.BOLD,
        color=COLOR_PRIMARY,
    )
    return ft.Container(
        bgcolor=ft.Colors.SURFACE_CONTAINER_HIGHEST,
        padding=PADDING_LOGO,
        border_radius=RADIUS_CARD,
        alignment=ft.Alignment.CENTER,
        content=logo,
    )

The texto parameter has a default value ("NotFlix"), so it can be called without arguments: build_logo(). In this version of the project it's barely used because the AppBar logo is defined directly in the router.
