Router

The brain of navigation. Controls which screen is shown and manages history. It's the file that connects all the views.

How navigation works in Flet

Flet doesn't use real URLs like a browser. Instead, it maintains a View stack. Think of it like a stack of cards: the card on top is what the user sees. When you navigate forward, a card is added on top. When you press back, the top card is removed.

how the View stack works
# On the start screen:
page.views = [profile_view]    # user sees profile

# Navigating to home:
page.views = [profile_view, home_view]    # user sees home

# Viewing a detail:
page.views = [profile_view, home_view, detail_view]    # user sees detail

# Pressing back:
page.views.pop()    # removes detail → user sees home

The base is always profile_view. Even if the user never goes back to it by clicking, it remains in the stack in case they press the physical back button multiple times.

The movie index — _MOVIE_INDEX

router.py — movie index
_MOVIE_INDEX: dict[str, Movie] = {m.show_id: m for m in load_movies()}

A dictionary that maps show_id → Movie. It's built once when the module is imported, using a dict comprehension (same as list comprehension but creates a dictionary).

When the route is /detail/s123, s123 is extracted and _MOVIE_INDEX.get("s123") is called. Looking up a dictionary is instant regardless of how many movies there are — much more efficient than scanning an entire list.

navigate_to() — the key function

router.py — navigate_to()
def navigate_to(page: ft.Page, route: str) -> None:
    page.views.clear()                     # clear the entire stack
    page.views.append(_view_profiles(page))  # always the base

    if route == "/home":
        page.views.append(_view_home(page))

    elif route.startswith("/detail/"):
        show_id = route.split("/detail/", 1)[-1]   # extract "s123"
        page.views.append(_view_home(page))
        page.views.append(_view_detail(page, show_id))

    page.update()   # redraw

Why clear and rebuild instead of just appending?

If you only appended, the stack would grow infinitely as the user navigates between screens. Clearing and rebuilding ensures the stack always has exactly the right depth for the current route.

Extracting the show_id from the route

route.split("/detail/", 1) splits the string "/detail/s123" into at most 2 parts: ["", "s123"]. The [-1] takes the last element, which is the ID. The 1 limits the splits to avoid breaking IDs that might contain the text "/detail/".

View builder functions

Each screen is wrapped in a ft.View, which is Flet's unit of a screen. This is where the route, background color, and optionally the AppBar are defined.

router.py — _view_home()
def _view_home(page: ft.Page) -> ft.View:
    return ft.View(
        route="/home",
        bgcolor=COLOR_BG,
        appbar=ft.AppBar(
            title=ft.Text("NOTFLIX", color="#E50914", weight=ft.FontWeight.BOLD),
            bgcolor="#0a0a0a",
            automatically_imply_leading=False,  # no automatic back button
        ),
        controls=[
            ft.Container(
                expand=True,
                padding=ft.Padding(16, 0, 16, 16),
                content=ft.SafeArea(expand=True, content=build_home_view(page)),
            )
        ],
    )

automatically_imply_leading=False disables the back arrow that Flet would automatically add to the AppBar when there's more than one view in the stack. On the home screen, that button is not wanted.

ft.SafeArea adds automatic padding to prevent content from being hidden by notches, cameras, or system bars on mobile.

setup_router() — the startup

router.py — setup_router()
def setup_router(page: ft.Page) -> None:
    def view_pop(e: ft.ViewPopEvent) -> None:
        if len(page.views) > 1:
            page.views.pop()   # remove the last view
            page.update()

    page.on_view_pop = view_pop    # physical back button
    navigate_to(page, "/")         # initial screen

page.on_view_pop is the event Flet fires when the user presses the physical back button (on Android) or the back gesture. The view_pop function removes the last view from the stack. The len > 1 check prevents removing the base view and ending up with an empty stack.

This function is called once from main.py when the app starts.
